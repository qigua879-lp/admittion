## 第 3 页：总体架构与主要设计进展

### PPT 建议内容

- 总体架构：
  - D-PHY 数字抽象输入
  - lane 数据整理与 CSI-2 包解析
  - 像素重组
  - FIFO/CDC 与 AXI 写入
  - 错误监测与状态输出
- 已初步实现的主要模块：
  - short / long packet parser
  - header ECC / payload CRC
  - frame / line sync
  - lane deskew / reorder
  - RAW8 / RAW10 / RGB888 / YUV422 像素重组
  - 基础 AXI 写入与 APB 状态接口

### 讲稿

目前系统采用“数字抽象输入、协议解析、像素重组、缓存写入和可靠性监测”的总体架构。

前端以 D-PHY 数字抽象信号作为输入，经过 lane 数据整理、多通道合并和 CSI-2 包解析后，提取帧、行和 payload 数据；中间完成 RAW8、RAW10、RGB888、YUV422 等格式的像素重组；后端通过 FIFO/CDC 和 AXI 写通路完成缓存与写入建模。

在模块实现方面，目前已初步实现 short packet 和 long packet 解析、header ECC、payload CRC、frame/line sync、多 lane deskew/reorder、像素重组、异步 FIFO、基础 AXI 写入和 APB 配置状态接口等模块。

整体上，当前 RTL 架构已经形成初步原型，能够支撑现阶段功能仿真和机制验证，但距离完整板级系统和 ASIC 后端实现仍有进一步工作。

## 第 4 页：可靠性机制与错误处理进展

### PPT 建议内容

- 针对问题：
  - 误码
  - 行帧错位
  - lane skew overflow
  - 写入端阻塞
- 阶段性实现的机制：
  - ECC / CRC 错误检测
  - 同步异常检测
  - 错误计数与状态输出
  - resync 恢复流程
  - 恢复后 clean frame 输出

### 讲稿

针对开题阶段提出的误码、行帧错位、数据溢出或欠读等问题，目前已经设计了初步的可靠性监测链路。

系统能够对 ECC 错误、CRC 错误、同步顺序异常和 lane skew overflow 等情况进行检测，并通过错误计数和状态输出保留异常信息。

在恢复机制方面，目前实现了基础 resync 控制流程，可以在检测到同步异常后触发清除和恢复过程，并在恢复后重新输出 clean frame。

现阶段该部分主要完成了基础机制闭合和基本功能验证，后续还需要进一步扩展到更多 lane 配置、更长时间运行以及更多异常组合条件下进行量化分析。

### 讲稿

目前已经搭建了模块级 testbench、wrapper 级系统测试、错误注入测试和部分参数扫描测试。

模块级测试主要验证 parser、FIFO、CRC、ECC、像素重组和 AXI writer 等局部功能；系统级测试主要观察从 CSI-2 输入到像素输出、AXI 写入以及错误信号传播的完整路径。

围绕当前已基本实现的内容，已经形成了一组代表性 testbench 和波形结果，主要覆盖协议解析、ECC/CRC 校验、多 lane 对齐、像素重组、系统主链路、缓存写入和错误恢复等部分。

从测试内容上看，当前验证并不是只对单个模块进行编译检查，而是围绕预期研究内容形成了初步对应关系。例如，short/long packet parser 用于验证 CSI-2 包字段解析和 payload 边界识别；header ECC 和 payload CRC 测试用于验证校验逻辑以及错误输出；lane deskew 和 lane reorder 测试用于验证多 lane 字节流的对齐与合并；RAW8、RAW10、RGB888 和 YUV422 相关测试用于验证不同数据格式的像素重组。

在系统级测试中，RAW8 wrapper smoke 和顶层 testbench 用于验证 CSI-2 输入到像素输出的基础闭环；AXI backpressure 和 AXI memory closure 测试用于观察写入端阻塞和写入闭环行为；resync recovery 和 clean frame 测试用于验证同步异常后的恢复流程。总体来看，这些测试结果能够说明当前协议解析、校验、多 lane 对齐、像素输出、基础写入和恢复链路已经具备阶段性验证依据。

在 FPGA wrapper 可综合性初步评估方面，当前报告对象是 `mipi_csi2_capture_fpga_wrapper`，不是最终板级完整系统。该 wrapper 版本约使用 618 个 LUT、1135 个 FF，BRAM 和 DSP 使用为 0；在当前占位时钟约束下，routed WNS 为 1.884 ns，TNS 为 0。

因此，这一部分更适合表述为“当前 RTL wrapper 原型具备初步综合和时序评估结果”，而不是“完整 FPGA 系统实现完成”。后续如果继续推进板级验证，还需要补充实际板级 IO 约束、输入输出 delay 和 bitstream 级 DRC 收敛。

## 第 6 页：当前边界与待完善工作

### PPT 建议内容

- 当前研究范围聚焦数字 RTL 接收前端
- 长时间稳定性和随机阻塞测试仍需补充
- AXI 写入已完成基础验证，后续需进一步结合完整 DDR 控制器系统
- FPGA 工程约束和 bitstream 级 DRC 仍需完善
- 后续还需继续整理资源、时序和验证矩阵等论文实验材料

### 讲稿

目前课题已经形成数字 RTL 接收前端的阶段性设计与仿真结果，但从后续论文和工程验证角度看，还需要继续完善以下几方面工作。

第一，当前研究范围聚焦 CSI-2 数字接收前端，输入侧采用 D-PHY 数字抽象接口。这一点与开题阶段的研究范围是一致的，后续主要围绕数字链路继续完善，而不把模拟 D-PHY 电气实现作为本阶段研究重点。

第二，现有仿真已经覆盖模块级测试、系统级测试和错误注入测试，后续还需要进一步补充更长时间运行、随机阻塞、更大帧尺寸和更复杂异常组合条件下的验证结果。

第三，AXI 写入目前已经完成基础写通路和写入端阻塞场景验证，后续需要进一步结合完整 DDR 控制器系统，完善从图像数据输入到存储端写入的工程化验证。

第四，Vivado 方面目前已有资源和时序结果，后续还需要继续完善 FPGA 工程约束，包括板级 IO 约束、时钟约束和 bitstream 级 DRC 问题分析。

第五，后续还需要进一步整理实验材料，包括系统架构图、验证矩阵、关键波形、资源利用率和时序结果表，为论文实验章节和后续答辩材料提供支撑。
