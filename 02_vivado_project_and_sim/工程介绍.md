# Vivado 工程及仿真文件说明

## 1. Vivado 工程内容

本目录下的 `vivado/` 来自原工程 `fpga/vivado/`，包含可重建工程的 TCL/PowerShell 脚本、已有 Vivado 工程工作目录、XSim smoke 仿真入口、正式波形脚本和综合实现报告。

| 文件/目录 | 说明 |
| --- | --- |
| `vivado/work/` | 已生成过的 Vivado 工程目录，包含 `.xpr` 工程文件 |
| `vivado/create_project.tcl` | 从源码重新创建 Vivado 工程 |
| `vivado/run_synth_impl.tcl` | 运行综合和实现 |
| `vivado/run_synth_route_direct.tcl` | 运行综合后直接执行 `opt/place/route/report/write_bitstream`，避开 Vivado Windows wrapper 链路卡住的问题 |
| `vivado/check_board_io_drc_v1.ps1` | 检查 board IO v1 的 bitstream、DRC 报告和 `NSTD-1/UCIO-1` 清零结果 |
| `vivado/run_full_build.ps1` | Windows 下一键构建入口 |
| `vivado/run_timing_cdc_v2_build.ps1` | 使用独立工程名重新跑 `timing_cdc_v2` 资源/时序报告 |
| `vivado/run_xsim_smoke.tcl` | XSim smoke 仿真 TCL |
| `vivado/run_xsim_smoke.ps1` | Windows 下 XSim smoke 仿真入口 |
| `vivado/build_one_xsim_snapshot.ps1` | 生成单个 XSim snapshot |
| `vivado/capture_one_xsim_wave.ps1` | 抓取正式波形截图 |
| `vivado/formal_wave_tcl/` | 正式波形的 TCL 配置 |
| `vivado/reports/` | Vivado 综合、实现、时序、资源报告 |
| `xdc/` | FPGA 约束文件 |

## 3. 工程使用方式

如果在 Vivado GUI 中查看，可优先打开：

```text
02_vivado_project_and_sim/vivado/work/mipi_csi2_capture_fpga_head_20260519/*.xpr
```

如果需要重新生成工程，可在原工程根目录或本交付包对应源码目录下参考：

```powershell
powershell -ExecutionPolicy Bypass -File fpga/vivado/run_full_build.ps1
```

如果只想看脚本化流程，重点看：

```text
vivado/create_project.tcl
vivado/run_synth_impl.tcl
vivado/run_impl_route_only.tcl
```

## 4. 我的工程说明

我在 Vivado 侧主要完成了 FPGA wrapper 的工程化验证，而不是最终落板。当前资源报告可以证明 RTL 能够被 Vivado 综合和实现，I/O 数量也通过 wrapper 做了收敛。`board_io_v1` 又补充了实验版 `LOC/IOSTANDARD/IO delay`，并生成了 bitstream，`NSTD-1/UCIO-1` 已在 bitgen DRC 中清零。需要注意的是，这仍是实验版 pinout，不是按真实板卡原理图签核后的上板约束。

## 5. timing_cdc_v2 更新说明

本次更新后，Vivado 结果新增了 `timing_cdc_v2` 版本：

| 项目 | 结果 |
| --- | ---: |
| Global routed WNS | `1.884 ns` |
| Global routed TNS | `0.000 ns` |
| `clk_byte` WNS | `2.476 ns` |
| `clk_sys` WNS | `1.884 ns` |
| LUT after implementation | `618` |
| FF after implementation | `1135` |
| Bonded IOB | `77 / 328` |

这个结果说明：补充 CDC 属性和 `clk_byte/clk_sys` 异步时钟组约束以后，当前占位时钟约束下 routed timing 已经满足。后续 `board_io_v1` 已进一步补齐实验版板级 `LOC/IOSTANDARD/IO delay`，并验证 `write_bitstream` 前 DRC 为 `0 Errors`。当前仍未完成的是按真实原理图的板级 pinout 签核和最终 timing closure。
