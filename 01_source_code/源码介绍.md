# 源码介绍

## 1. 源码范围

本目录中的 `rtl/` 是本次交付的主要 RTL 源码，代码语言以 Verilog/SystemVerilog 为主，DUT 与 testbench 分开放置。当前阶段只实现数字逻辑，不包含真实模拟 D-PHY 电气模型。

主要顶层为：

| 顶层 | 位置 | 作用 |
| --- | --- | --- |
| `mipi_csi2_capture_top` | `rtl/top/mipi_csi2_capture_top.sv` | 数字 CSI-2 接收系统主顶层 |
| `mipi_csi2_capture_fpga_wrapper` | `rtl/top/mipi_csi2_capture_fpga_wrapper.sv` | 面向 FPGA 验证的封装顶层 |
| `fpga_apb_boot_cfg` | `rtl/top/fpga_apb_boot_cfg.sv` | FPGA wrapper 下的简单 APB 启动配置 |

## 2. 模块划分

| 模块目录 | 主要文件 | 说明 |
| --- | --- | --- |
| `rtl/reg_if/` | `cfg_reg_if_apb.sv` | APB 配置与状态寄存器接口 |
| `rtl/phy_adapter/` | `phy_digital_adapter.sv` | D-PHY 数字抽象输入适配，不实现模拟电气 |
| `rtl/csi2_rx/` | parser、ECC、CRC、frame sync、lane merge | CSI-2 包解析、lane 对齐、Header ECC、Payload CRC、帧行同步 |
| `rtl/pixel/` | `raw8_unpack.sv`、`raw10_unpack.sv`、`rgb888_unpack.sv`、`yuv422_unpack.sv` | RAW8/RAW10/RGB888/YUV422 像素重组 |
| `rtl/reliability/` | `err_classifier.sv`、`err_frame_line_logger.sv`、`retry_request_ctrl.sv`、`resync_ctrl_fsm.sv`、`degrade_recover_fsm.sv` | 错误分类、帧行记录、自动重采集请求、重同步和降级恢复 |
| `rtl/buffer/` | `async_fifo.sv` | FIFO/CDC 缓冲基础模块 |

## 3. 我的源码设计思路

我把系统拆成“输入适配、协议解析、像素输出、可靠性处理、缓存跨时钟、AXI 写入”几条相对独立的路径。这样做的好处是每个模块都能单独写 testbench，也方便后面替换真实 D-PHY 或接入更完整的 DDR 控制器。

所有时序逻辑都按同步设计处理，复位统一使用低有效 `rst_n`。接口命名遵守工程要求，例如时钟用 `clk_xxx`，有效信号用 `xxx_valid`，就绪信号用 `xxx_ready`，错误信号用 `err_xxx`。

## 4. 本次代码层改动

| 文件 | 改动 | 影响 |
| --- | --- | --- |
| `rtl/buffer/async_fifo.sv` | 给 Gray pointer 两级同步寄存器添加 `ASYNC_REG` 和 `SHREG_EXTRACT=NO` 属性 | 让 FPGA 工具正确识别 CDC 同步结构，避免同步器被移位寄存器优化 |
| `rtl/top/mipi_csi2_capture_top.sv` | 给配置同步、lane overflow toggle、resync toggle 等跨域同步寄存器添加相同属性 | 强化 `clk_byte` 与 `clk_sys` 之间的 CDC 实现说明 |
| `rtl/axi/axi_write_null_slave.sv` | 增加 `MEM_ADDR_WIDTH` 参数、内部 32-bit memory、WSTRB byte 写入和 overflow response | 让 wrapper 内部 AXI sink 不只是“吃掉写事务”，还可以回读检查写入内容 |
| `rtl/reliability/retry_request_ctrl.sv` | 新增 retry request v1，错误发生时锁存 frame/line/VC/DT 并输出一拍请求 | 支撑“错误帧/行定位 + 帧级或行级重采集请求”的论文创新点 |
| `rtl/reg_if/cfg_reg_if_apb.sv` | 新增 `LAST_ERR_FRAME/LINE`、`RETRY_STATUS/FRAME/LINE` 与 `CTRL[3] retry_ack` | 软件可直接读取最近错误位置和重采集请求位置 |
| `fpga/xdc/top_constraints.xdc` | 新增 `set_clock_groups -asynchronous`，声明 `clk_byte` 与 `clk_sys` 异步 | 配合现有 FIFO/toggle synchronizer 结构，修正 Vivado 对跨域路径的保守分析 |

这次改动没有改变顶层接口风格，也没有改变像素输出和 ready/valid 的基本协议。主要目的是把已有 CDC 结构表达得更清楚，并增加 AXI 写入结果可验证性。

## 5. 边界条件

| 边界 | 当前处理 |
| --- | --- |
| 真实 D-PHY 电气 | 当前没有实现，只使用数字 lane 输入 |
| lane 数 | 已支持 `1 / 2 / 4 lane` 配置，其中 `2 lane` 验证最充分 |
| 像素格式 | 已覆盖 RAW8、RAW10、RGB888、YUV422 |
| AXI | 当前重点是 AXI 写通路，不包含完整 DDR 系统 |
| ASIC | 当前只有评估说明和占位，不是完整 ASIC 后端交付 |

## 6. 自检方法

源码自检主要通过 `tb/tests/` 中的模块级和 wrapper 级 testbench 完成。编译入口可以参考 `04_tb_tests/compile.f` 和 `02_vivado_project_and_sim/sim/vcs/` 下的脚本。
