# 网表级文件说明

## 1. 文件来源

网表级文件来自 Vivado 报告目录，目前保留两个版本：

```text
06_netlist_level_files/mipi_csi2_capture_fpga_head_20260519/
06_netlist_level_files/mipi_csi2_capture_fpga_timing_cdc_v2/
```

综合对象为：

```text
mipi_csi2_capture_fpga_wrapper
```

目标器件为：

```text
xczu9eg-ffvb1156-2-e
```

## 2. 已包含文件

| 文件 | 说明 |
| --- | --- |
| `*_synth_netlist.v` | Vivado 综合后 Verilog 网表 |
| `*_synth.edf` | EDIF 网表 |
| `*_synth.xdc` | 综合导出的约束 |
| `post_synth.dcp` | 综合后 design checkpoint |
| `post_impl.dcp` | 实现后 design checkpoint |
| `synth_utilization.rpt` | 综合资源报告 |
| `impl_utilization.rpt` | 实现资源报告 |
| `synth_timing_summary.rpt` | 综合时序报告 |
| `impl_timing_summary.rpt` | 实现后时序报告 |
| `impl_power.rpt` | 功耗估计报告 |
| `impl_drc.rpt` | 实现后 DRC 报告 |

## 3. 资源结果摘要

| 版本 | LUT 综合后 | LUT 实现后 | FF 综合后 | FF 实现后 | BRAM | DSP | Bonded IOB |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| `timing_cdc_v1/head_20260519` | 541 | 530 | 818 | 819 | 0 | 0 | 59 / 328 |
| `timing_cdc_v2` | 634 | 618 | 1165 | 1135 | 0 | 0 | 77 / 328 |

## 4. 时序结果摘要

| 指标 | 数值 | 说明 |
| --- | ---: | --- |
| 版本 | Global routed WNS | Global routed TNS | 说明 |
| --- | ---: | ---: | --- |
| `timing_cdc_v1/head_20260519` | -2.725 ns | -327.762 ns | 保留为 CDC 约束修正前的问题基线 |
| `timing_cdc_v2` | 1.884 ns | 0.000 ns | 当前占位时钟约束下 routed timing 已满足 |

`timing_cdc_v2` 的意义是：在 RTL CDC 属性和 XDC 异步时钟组约束补充后，原先 `clk_byte -> clk_sys` 的负 slack 不再作为同步路径违例出现。

仍需注意：bitstream 还没有最终完成，因为板级 `LOC` 和 `IOSTANDARD` 缺失导致 `NSTD-1` 和 `UCIO-1` DRC 保留。

## 5. ASIC 说明

`asic_notes/` 中是 Genus、Innovus 和 reports 的说明文件，目前属于 ASIC 评估占位。当前交付没有包含基于工艺库的 ASIC 门级网表、SDF 反标文件或完整 PnR 数据。因此如果老师问“网表级文件”，我目前能交付的是 Vivado FPGA 侧综合/实现网表和 DCP，不是 ASIC signoff 网表。
