# 接口表格及说明

## 1. 顶层接口

顶层接口以 `mipi_csi2_capture_top` 为主，FPGA 验证时通过 `mipi_csi2_capture_fpga_wrapper` 做封装。

### 时钟与复位

| 信号 | 方向 | 位宽 | 时钟域 | 说明 |
| --- | --- | ---: | --- | --- |
| `clk_sys` | input | 1 | system | 协议控制、状态统计和像素调试输出域 |
| `clk_byte` | input | 1 | byte | 数字 lane 接收域 |
| `clk_axi` | input | 1 | AXI | AXI 写通路域 |
| `clk_ddr` | input | 1 | DDR | DDR 相关预留域 |
| `rst_n` | input | 1 | all | 低有效同步复位 |

## 2. 接口使用说明

我的设计中，`clk_byte` 接收 lane 侧数据，经过协议解析和 FIFO/CDC 后进入 `clk_sys` 域，再输出像素调试信号和状态信号。AXI 写入单独使用 `clk_axi` 域，方便后面接 DDR 或 FPGA 片上存储系统。

为了降低早期实现复杂度，配置接口使用 APB，而不是一开始就使用 AXI-Lite。这样做更适合当前阶段的 RTL 验证，也方便 testbench 直接写寄存器。
