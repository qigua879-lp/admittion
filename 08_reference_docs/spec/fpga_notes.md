# FPGA / Vivado Notes

## Scope

This stage adds a Vivado project/script skeleton only. It does not implement `mipi_csi2_capture_top`, does not wire lower-level RTL into a final top, and does not modify any previously verified RTL or testbench interface.

For non-Vivado smoke and full RTL regression entry points, see `docs/spec/regression_plan.md`.

## Added Files

| File | Role |
| --- | --- |
| `fpga/vivado/create_project.tcl` | Creates a Vivado project, adds RTL sources under `rtl/`, adds XDC constraints under `fpga/xdc/`, and sets the requested top when it exists. |
| `fpga/vivado/run_synth_impl.tcl` | Opens the generated Vivado project, checks that the requested top module exists, runs synthesis and implementation, and exports reports. |
| `fpga/xdc/top_constraints.xdc` | Provides clock, IO standard, pin, CDC, and reset constraint placeholders without assuming a board. |
| `docs/spec/fpga_notes.md` | Documents script usage, expected output layout, manual board information, checks, and known limitations. |

## Expected Directory Layout

```text
fpga/
  vivado/
    create_project.tcl
    run_synth_impl.tcl
    work/
      <project_name>/
        <project_name>.xpr
    reports/
      <project_name>/
        synth_utilization.rpt
        synth_timing_summary.rpt
        synth_cdc.rpt
        impl_utilization.rpt
        impl_timing_summary.rpt
        impl_power.rpt
        impl_drc.rpt
        post_impl.dcp
        *.bit
  xdc/
    top_constraints.xdc
```

The `work/` and `reports/` directories are generated by the Tcl scripts and are not required to exist before running Vivado.

## Create Project

The create script requires a real Vivado part name. The repository intentionally does not provide a default device.

```sh
vivado -mode batch -source fpga/vivado/create_project.tcl \
  -tclargs <fpga_part> mipi_csi2_capture mipi_csi2_capture_top
```

You can also provide the part through the environment:

```sh
export FPGA_PART=<fpga_part>
vivado -mode batch -source fpga/vivado/create_project.tcl
```

If `mipi_csi2_capture_top` has not been implemented yet, the script still creates a source/constraint project and prints a warning. This is intentional for the current stage.

## Run Synthesis And Implementation

Run this only after top integration RTL exists:

```sh
vivado -mode batch -source fpga/vivado/run_synth_impl.tcl \
  -tclargs mipi_csi2_capture mipi_csi2_capture_top 8
```

Arguments:

| Argument | Meaning |
| --- | --- |
| `project_name` | Project directory name under `fpga/vivado/work/`. |
| `top_module` | Expected final synthesis top. |
| `jobs` | Vivado parallel job count. Can also be set with `VIVADO_JOBS`. |

The run script stops before synthesis if the requested top module is not found in the project sources.

## Required Manual Board Information

Before a real FPGA run, fill in:

| Item | Where |
| --- | --- |
| FPGA part, such as `xc...-...` | `create_project.tcl` command argument or `FPGA_PART` env var |
| Optional Vivado board part | Add explicitly after board selection if board flow is required |
| `clk_sys` period | `top_constraints.xdc` |
| `clk_byte` period | `top_constraints.xdc` |
| `clk_axi` period | `top_constraints.xdc` |
| `clk_ddr` period | `top_constraints.xdc` |
| Package pin mapping | `top_constraints.xdc` |
| IO standards and bank voltages | `top_constraints.xdc` |
| Final clock relationship constraints | `top_constraints.xdc` after CDC/top integration |
| Which debug/status/pixel ports are external | Top integration spec and XDC |

## Constraint Policy

- The XDC file creates clocks only when matching top-level ports exist.
- IO standards are disabled by default with `DEFAULT_IOSTANDARD` set to an empty string.
- Pin constraints are comments only until a real package and schematic are selected.
- `rst_n` remains synchronous by project rule; no async reset false path is added.
- `clk_byte` and `clk_sys` are declared asynchronous because the current top crosses that boundary through `u_byte_to_sys_fifo` and explicit toggle/config synchronizers.
- Other clock relationships remain board/integration dependent until the final clock tree and DDR/AXI integration are frozen.

## Self-Check

Minimum checks for this stage:

1. Tcl/XDC files are syntactically complete under Tcl parsing.
2. `create_project.tcl` rejects a missing FPGA part with a clear message.
3. `run_synth_impl.tcl` rejects a missing top integration module with a clear message.
4. No RTL or testbench interfaces are modified.

## Known Limitations

- The scripts do not run a real Vivado build until a concrete FPGA part and final top integration RTL exist.
- Timing constraints are placeholders and must be updated to match board clocks.
- No DDR controller, board automation, ILA core, or IP generation is added in this stage.
