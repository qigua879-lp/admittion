# CDC Timing v2 Notes

## Version

- Previous baseline: `timing_cdc_v1`
- Current update: `timing_cdc_v2`
- Date: 2026-05-20

## Preserved Issue

`timing_cdc_v1` kept the routed global timing issue as an explicit baseline:

- `WNS = -2.725 ns`
- `TNS = -327.762 ns`
- Main violated clock pair: `clk_byte -> clk_sys`
- Worst path:
  - `u_byte_to_sys_fifo.mem_reg_0_15_0_5/RAMD_D1`
  - `u_payload_crc_checker.crc_error_reg/D`

Root cause: Vivado analyzed the asynchronous byte-domain FIFO memory output path as a normal synchronous path into `clk_sys`.

## RTL Changes

- `rtl/buffer/async_fifo.sv`
  - Adds `ASYNC_REG` and `SHREG_EXTRACT=NO` attributes to Gray pointer synchronizer stages.
- `rtl/top/mipi_csi2_capture_top.sv`
  - Adds the same attributes to top-level CDC synchronizer stages for:
    - byte-domain lane configuration sampling
    - byte-to-system lane overflow event toggle
    - system-to-byte resync event toggle

No top-level ports, module interfaces, or ready/valid behavior are changed.

## Constraint Changes

- `fpga/xdc/top_constraints.xdc`
  - Declares `clk_byte` and `clk_sys` as asynchronous clock groups.

This exception is justified by the current top-level CDC structure:

- payload byte stream crosses through `u_byte_to_sys_fifo`
- lane overflow crosses through a toggle synchronizer
- resync request crosses back through a toggle synchronizer
- lane configuration values are synchronized into the byte domain and should change only during idle or reset/recovery windows

## Boundary Conditions

- The `clk_byte`/`clk_sys` clock group exception must be revisited if any new direct combinational or registered path is added between the two domains.
- Multi-bit configuration values are synchronized for FPGA implementation robustness, but software should update them only while capture is idle or during controlled recovery.
- The change itself does not resolve board-level `LOC`, `IOSTANDARD`, input/output delay, or DDR integration constraints. The later `board_io_v1` run adds a lab placeholder pinout and clears bitstream-blocking `NSTD-1/UCIO-1`, but it is still not a schematic-verified board signoff.

## Self-Check

Recommended checks after this update:

1. Re-run Vivado implementation and confirm the previous global worst pair `clk_byte -> clk_sys` is no longer reported as a synchronous violation.
2. Confirm same-domain timing still meets the placeholder clocks:
   - `clk_sys`
   - `clk_byte`
   - `clk_axi`
3. Run CDC reporting and inspect any remaining direct crossings between `clk_byte` and `clk_sys`.
4. Run the existing FIFO and wrapper smoke tests:
   - `tb_async_fifo`
   - `tb_fpga_wrapper_raw8_smoke`
   - `tb_fpga_wrapper_resync_clean_frame`

Vivado rerun command:

```powershell
powershell -ExecutionPolicy Bypass -File fpga/vivado/run_timing_cdc_v2_build.ps1 -Jobs 4
```

This uses project name `mipi_csi2_capture_fpga_timing_cdc_v2` so that the `timing_cdc_v1` reports remain untouched.

Implementation flow note:

- The stable rerun path for `timing_cdc_v2` uses a direct `opt/place/route/report` Tcl flow in the current batch session.
- This avoids the Vivado Windows WSH/cscript wrapper chain generated by `launch_runs impl_1`, which was observed to stall before producing route-stage logs on this machine.

## Observed Result

Direct routed implementation was completed on May 20, 2026 with reports captured under:

- `fpga/vivado/reports/mipi_csi2_capture_fpga_timing_cdc_v2/`

Key observations:

- Global routed timing:
  - `WNS = 1.884 ns`
  - `TNS = 0.000 ns`
- Report status:
  - `All user specified timing constraints are met.`
- Same-domain timing:
  - `clk_byte WNS = 2.476 ns`
  - `clk_sys WNS = 1.884 ns`
- Cross-domain timing table no longer reports `clk_byte -> clk_sys` as a failing synchronous pair.

Remaining limitation at the `timing_cdc_v2` stage:

- `write_bitstream` still fails because the placeholder XDC intentionally leaves board-level `LOC` and `IOSTANDARD` unspecified.
- Routed DRC therefore keeps:
  - `NSTD-1`
  - `UCIO-1`

Superseding board-IO result:

- `board_io_v1` adds `board_lab_placeholder_v1.xdc` plus IO delay constraints.
- `write_bitstream` DRC is `0 Errors`.
- `impl_drc.rpt` reports `Violations found: 0`.
- Bitstream is generated as `mipi_csi2_capture_board_io_v1_clkfix5.bit`.

## Known Limitations

- `timing_cdc_v2` is an FPGA timing-constraint and CDC-annotation update, not a full board timing closure.
- `board_io_v1` clears bitstream-blocking IO DRC with a lab placeholder pinout.
- Final schematic pinout, bank voltage signoff, DDR controller timing, and timing closure remain open.
