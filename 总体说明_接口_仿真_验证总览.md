# 总体说明、接口、仿真与验证总览

本文档把当前交付包里分散的总体说明、接口定义、仿真工程和验证结果整合成一个统一入口，便于老师、评审或后续维护者快速理解工程现状。

## 1. 总体说明

本项目当前完成的是一个面向数字 RTL 的 `MIPI CSI-2` 图像采集前端原型，重点覆盖：

- `CSI-2` 包解析
- `1 / 2 / 4 lane` 接收、deskew 与 reorder
- `ECC / CRC` 检错
- `frame / line` 同步
- `RAW8 / RAW10 / RGB888 / YUV422` 像素重组
- 基础恢复链路，包括 `resync`、异常计数与恢复后 clean frame 输出
- `FIFO / CDC`
- `AXI` 写入通路
- 轻量预处理模块

当前最稳妥的工程口径是：

> 已完成面向数字 RTL 的 `MIPI CSI-2` 图像采集前端主体设计与系统级验证，核心功能、基础恢复能力和主要边界条件均已有可追溯证据；当前结果可以支撑“数字前端工程原型”结论，但还不是完整产品态，也不是 `ASIC-ready` 或最终板级 signoff 状态。

不在当前完成范围内的部分主要有：

- 真实模拟 `D-PHY` 电气实现
- 最终板级 `LOC / IOSTANDARD / IO delay` 完整收敛
- 全覆盖长时间 soak、更多 corner 组合和等强度 lane 全配置异常验证

## 2. 总体说明中的关键对比

### 2.1 像素格式对比

当前已经完成系统级闭环验证的格式有 `RAW8 / RAW10 / RGB888 / YUV422`。

| Format | `init_to_frame` | `frame_to_first_pixel` | `frame_to_end` | `first_to_last_pixel` | `pixel_valid_cycles` | Full-frame closure |
| --- | ---: | ---: | ---: | ---: | ---: | --- |
| `RAW8` | `14` | `16` | `34` | `3` | `4` | Yes |
| `RAW10` | `16` | `20` | `38` | `3` | `4` | Yes |
| `RGB888` | `14` | `18` | `42` | `9` | `4` | Yes |
| `YUV422` | `14` | `19` | `40` | `7` | `4` | Yes |

可以直接得出的比较结论：

- `RAW8` 是当前最短的 baseline 主链路。
- `RAW10` 与 `RAW8` 像素输出跨度一致，但前端整理时延更高。
- `RGB888` 的整帧跨度最大，`YUV422` 介于 `RAW10` 与 `RGB888` 之间。

### 2.2 Lane 配置对比

当前系统已经有 `1 / 2 / 4 lane` 的 wrapper 级证据，但验证强度并不完全相同。

| `LANE_NUM` | 当前证据强度 | 结论 |
| --- | --- | --- |
| `1` | wrapper smoke | 已实现基本闭环 |
| `2` | 主验证路径 | 证据最充分 |
| `4` | wrapper smoke | 已实现基本闭环 |

最稳妥的表述是：`1 / 2 / 4 lane` 配置能力已经实现，且三种配置都具备至少一条 wrapper 级闭环证据，但 `2 lane` 仍是当前最强主验证路径。

### 2.3 Buffer / 背压趋势对比

`BYTE FIFO` 深度、`AXI FIFO` 深度和背压释放延迟已经做过基础扫描。当前可直接引用的工程结论有：

- 在既定 `16-pixel` 样例下，多数组合都能保持 `exp=act`。
- `BYTE FIFO` 变深会提高占用上限，但当前这个 sweep 中没有改变 `AXI busy duration=4` 的基础行为。
- 连续流强化场景已经观察到浅 `AXI FIFO` 配置存在失稳边界，因此当前工程默认值不宜继续下探。

### 2.4 恢复能力对比

当前恢复链路不是“只会报错”，而是已经有恢复后继续工作的证据：

- `CRC / ECC` 错误注入后能观测到 `err_*` 与错误计数跳变
- `sync error -> resync_req -> busy -> clear -> done` 链路已闭环
- 恢复后 clean frame 输出、恢复后 multiframe 连续输出都已闭环

## 3. 接口总览

顶层以 `mipi_csi2_capture_top` 为主，FPGA 侧验证使用 `mipi_csi2_capture_fpga_wrapper` 封装。

### 3.1 时钟与复位

| 信号 | 方向 | 时钟域 | 说明 |
| --- | --- | --- | --- |
| `clk_sys` | input | system | 协议控制、状态统计、像素调试输出 |
| `clk_byte` | input | byte | lane 数据接收域 |
| `clk_axi` | input | AXI | `AXI` 写通路域 |
| `clk_ddr` | input | DDR | DDR 预留域 |
| `rst_n` | input | all | 低有效同步复位 |

### 3.2 输入侧接口

输入侧保留数字抽象 `PHY` 信号：

- `lane_data_0..3`
- `lane_valid_0..3`
- `hs_mode`
- `lp_mode`

当前仓库聚焦数字前端，所以这些信号用于抽象 `D-PHY` 输出，而不是模拟电气级建模。

### 3.3 配置与存储接口

| 接口 | 作用 | 选择原因 |
| --- | --- | --- |
| `APB` 配置接口 | 寄存器读写、状态访问 | 当前阶段比 `AXI-Lite` 更轻，便于 RTL 与 testbench 验证 |
| `AXI` 写接口 | 写地址、写数据、写响应 | 作为后端写入 DDR / 片上存储系统的主通路 |

### 3.4 调试与状态输出

当前可直接用于观测和验证的信号包括：

- `frame_start_o / frame_end_o`
- `line_start_o / line_end_o`
- `pixel_data_o / pixel_valid_o / pixel_sof_o / pixel_sol_o`
- `err_ecc_o / err_crc_o / err_sync_o`
- `frame_cnt_o / line_cnt_o / err_cnt_ecc_o / err_cnt_crc_o`

这些输出也是当前 testbench 自检、正式波形截图和论文插图的主要观测点。

## 4. 仿真与工程组织

### 4.1 仿真分层

当前验证按四层组织：

| 层级 | 目标 | 典型入口 |
| --- | --- | --- |
| 模块级 | 验证 parser / FIFO / CRC / deskew / writer 等局部行为 | `04_tb_tests/tb/tests/*.sv` |
| wrapper 级系统验证 | 验证真实系统路径下 `FS/LS/payload/LE/FE -> pixel -> AXI` 闭环 | `tb_fpga_wrapper_*` |
| 参数扫描 | 比较 lane、buffer、背压、连续流等边界趋势 | `02_vivado_project_and_sim/scripts/*.ps1` |
| Vivado / XSim | 生成正式波形截图和资源时序结果 | `02_vivado_project_and_sim/vivado/*` |

### 4.2 关键仿真目录

| 目录 | 内容 |
| --- | --- |
| `04_tb_tests/` | `SystemVerilog` testbench、模型、scoreboard、说明文档 |
| `02_vivado_project_and_sim/sim/` | filelist、仿真入口、波形目录说明 |
| `02_vivado_project_and_sim/scripts/` | 参数扫描、批量重验证、正式波形抓图脚本 |
| `02_vivado_project_and_sim/vivado/` | `xvlog / xelab / xsim` 相关脚本、正式波形 `Tcl`、综合实现脚本 |

### 4.3 工程化说明

Vivado 侧当前完成的是“工程化验证”和“实现评估”，而不是最终落板。  
比较新的 `timing_cdc_v2` 结果说明，在补充 `CDC` 属性和 `clk_byte / clk_sys` 异步时钟组之后，当前占位约束下 routed timing 已满足：

| 项目 | 结果 |
| --- | ---: |
| Global routed `WNS` | `1.884 ns` |
| Global routed `TNS` | `0.000 ns` |
| `clk_byte` `WNS` | `2.476 ns` |
| `clk_sys` `WNS` | `1.884 ns` |

但这个结果仍不能直接等价成最终 bitstream signoff，因为板级约束还没有完整补齐。

## 6. 当前边界与引用注意事项

对外说明时，建议保留下面这些边界：

1. 当前聚焦数字前端，不包含真实模拟 `D-PHY`。
2. `2 lane` 是主验证路径，`1 lane / 4 lane` 目前仍偏 smoke 补证。
3. 一部分参数扫描用于观察趋势，不等价于最终吞吐极限。
4. Vivado 结果应表述为“工程原型实现评估”，不是最终板级 signoff。
5. 正式波形图用于展示关键机制，最终结论仍以 testbench 自检结果和 `.md` 结果表为准。
