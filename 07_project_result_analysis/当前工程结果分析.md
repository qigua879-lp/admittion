# 当前工程结果分析

## 1. 预期目标/指标

本课题预期完成一个基于 MIPI CSI-2 协议的高速高可靠图像采集前端系统，重点目标如下：

| 预期目标 | 指标或表现形式 |
| --- | --- |
| D-PHY 数字抽象输入 | 支持多 lane 数字输入和 HS/LP 模式指示 |
| CSI-2 包解析 | 支持 short packet、long packet、FS/FE/LS/LE |
| 多 lane 对齐与重排序 | 支持 lane deskew、lane reorder、overflow 检测 |
| ECC/CRC 校验 | Header ECC 和 Payload CRC 错误检测 |
| 帧/行同步 | 输出 frame/line marker 和计数 |
| 像素重组 | 支持 RAW8、RAW10、RGB888、YUV422 |
| 可靠性 | 错误统计、重同步、恢复后继续工作 |
| FIFO/CDC/缓存 | 支持 byte/sys/AXI 跨域缓冲 |
| AXI 写入 | 实现 AXI write path |
| AXI 写入可验证性 | AXI 写入后可以通过 memory readback 与 scoreboard 校验 |
| 轻量预处理 | 支持亮度、对比度、灰度平衡和 bypass |
| 验证 | 有模块级 TB、系统级 TB、错误注入、scoreboard |
| Vivado | 可综合、可实现，有资源和时序报告 |
| 网表级 | 提供 FPGA 综合后网表、EDF、DCP 和报告 |

## 2. 已完成目标/指标

| 已完成项 | 当前证据 |
| --- | --- |
| RTL 主链路 | `rtl/` 下已包含 top、parser、pixel、reliability、buffer、AXI、preprocess |
| 多格式像素 | RAW8、RAW10、RGB888、YUV422 均有 wrapper 级结果 |
| lane 配置 | 1/2/4 lane 已有闭环证据，其中 2 lane 最充分 |
| 多 lane soak | 1/2/4 lane 已新增 RAW8 多帧多行 soak 结果 |
| ECC/CRC | 有模块级和 wrapper 级错误注入测试 |
| resync 恢复 | 有恢复后 clean frame 和 multiframe/backpressure 结果 |
| AXI 背压 | 有 AW/W stall 场景和连续流压力扫描 |
| lane skew | 已得到 `DESKEW_DEPTH` 对应容忍窗口 |
| 多帧稳定性 | RAW8 多帧多行 `mismatch=0` |
| AXI 写入闭环 | `RAW8 -> AXI write -> memory readback -> scoreboard` 已覆盖 1/2/4 lane |
| Vivado 资源 | `timing_cdc_v2` 实现后约 `618 LUT / 1135 FF / 0 BRAM / 0 DSP` |

| 网表文件 | 已有 Verilog netlist、EDF、post_synth/post_impl DCP，且新增 `timing_cdc_v2` 版本 |

## 3. 未完成目标/指标

| 未完成或不足 | 当前原因 |
| --- | --- |
| 真实模拟 D-PHY | 课题当前阶段限定为数字逻辑，没有做模拟电气 |
| 最终板级 bitstream signoff | 实验版 LOC/IOSTANDARD/IO delay 已补齐并清掉 bitstream DRC；真实上板仍缺原理图 pinout、bank VCCO 和外设连接确认 |
| timing closure | `board_io_v1` 可生成 bitstream，但 `impl_timing_summary.rpt` 仍有负 slack，需要后续收敛 |
| lane1/lane4 错误注入深度 | 已有 smoke、soak、AXI readback，但异常注入覆盖不如 lane2 充分 |
| 超长时间稳定性 | 已有 32/48 帧 soak，但还没有小时级或板级长稳 |
| ASIC signoff | 目前只有 ASIC 评估占位，没有工艺库门级网表/SDF |
| DDR 完整系统 | 当前实现 AXI 写路径，还没有完整 DDR 控制器系统级集成 |

## 4. 改进策略

| 问题 | 改进策略 |
| --- | --- |
| 板级 bitstream DRC | 已有实验版 `LOC/IOSTANDARD/IO delay`，继续把 lab pinout 替换为真实板卡 pinout 并做 timing closure |

| 长时间稳定性不足 | 增加更长帧数、更长行长度和随机 backpressure 的 soak 测试 |

| ASIC 评估不足 | 增加 lint、CDC、约束整理、Genus 综合脚本和工艺库相关报告 |

## 5. 我的阶段性总结

从学生项目角度看，我认为当前工程已经完成了核心数字链路的搭建和验证。它已经不是只有模块代码的状态，而是有顶层 wrapper、系统 testbench、错误注入、scoreboard、Vivado 综合实现报告和网表级文件。

不足也比较明确：当前设计还没有进入最终板级或 ASIC signoff 阶段。CDC/timing 的主要问题已经通过 `timing_cdc_v2` 得到改善，实验版板级 IO DRC 也已通过 `board_io_v1` 清零；但真实板卡 pinout、长时间压力测试、lane1/lane4 等强度错误注入覆盖、DDR 实系统接入还需要继续完善。

所以我会把当前成果定位为：

```text
已完成 MIPI CSI-2 数字图像采集前端 RTL 原型和系统级验证闭环，timing_cdc_v2 下 FPGA routed timing 已满足占位时钟约束；board_io_v1 下实验版 LOC/IOSTANDARD/IO delay 已补齐并生成 bitstream，NSTD-1/UCIO-1 已清零。后续重点是真实板卡 pinout 签核、timing closure、长期稳定性测试和更完整工程集成。
```
