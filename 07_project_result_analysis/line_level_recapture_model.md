# 行级选择性重采集 — 内存/可靠性量化模型（T1）

> 本文件是创新点“方法贡献”章节的可推导基础。结论：在单向 MIPI CSI-2 链路上，
> 当**请求往返窗口跨越的行数远小于整帧行数**且**单行残余失效概率足够小**时，
> 行级选择性重采集相比“整帧重传 / 整帧丢弃”，在**内存开销**与**有效帧率**之间
> 取得更优折中；本文给出三种策略各自的闭式表达、可行性条件与量化边界。
>
> 对应交接文档 §3 命题、§4 任务 T1。模型与 RTL 解耦，可直接写入论文方法章。
> 数值脚本：[`tools/recapture_model.py`](../tools/recapture_model.py)。

## 1. 符号与系统参数

| 符号 | 含义 | 代表取值（工程默认） |
|---|---|---|
| `W` | 每行像素数 | 1920 |
| `H` | 每帧行数 | 1080 |
| `bpp` | 每像素打包字节数 | RAW10 → 1.25（RAW8=1，YUV422=2，RGB888=3） |
| `L = W·bpp` | 每行有效载荷字节 | 1920×1.25 = **2400 B** |
| `F = H·L` | 每帧有效载荷字节 | 1080×2400 = **2 592 000 B ≈ 2.47 MiB** |
| `f0` | 标称帧率 (fps) | 30 |
| `T_f = 1/f0` | 帧周期 | 33.3 ms |
| `T_l = T_f/H` | 行周期 | 30.9 µs |
| `BER` | 链路位错误率 | 扫描变量 |
| `p` | 单行经 ECC/CRC 后**残余失效概率**（该行被判坏） | `p ≈ 1-(1-BER)^{8L}`（小 BER 时 ≈ `8L·BER`） |
| `p_f = 1-(1-p)^H` | 整帧含≥1坏行的概率 | 小 p 时 ≈ `H·p` |
| `τ_rt` | 错误检出→上游重发到达的请求往返时延 | 由上游缓存/控制路径决定 |
| `D = ⌈τ_rt/T_l⌉` | 往返窗口跨越的行数 | 典型 4–16 |
| `h ≥ 1` | 链路带宽余量系数（`R_link = h·F·f0`） | 决定能否“追帧” |

> 说明：CSI-2 长包载荷由包尾校验/CRC 保护、包头由 ECC 保护，故此处把“单行是否可
> 用”抽象为残余失效概率 `p`，既可由 `BER` 推得，也可直接作为输入扫描。

## 2. 三种恢复策略的闭式模型

### 策略 A — 行级选择性重采集（本项目创新点）
只重发被判坏的行，往返窗口内上游保留可重发的行缓存。

- **重发开销**：单行直到成功的期望传输次数 `1/(1-p)`，每帧期望额外重传行数
  `H·p/(1-p)`，即额外链路字节 `F·p/(1-p)`。
- **维持满帧率条件**：`h ≥ 1/(1-p)`（≈ `1+p`，余量极小即可）。满足时 `f_A = f0`
  且坏行全部恢复。
- **内存开销**：上游只需保留往返窗口内仍可能被重请求的行：
  **`ΔM_A = D·L`**（RX 侧同阶有界）。
- **恢复时延**：单坏行 `= τ_rt = D·T_l`，**与 H 无关**。

### 策略 B — 整帧重传
任一行坏即重传整帧。

- **重发开销**：每帧期望传输次数 `1/(1-p_f)`，额外字节 `F·p_f/(1-p_f)`。
- **维持满帧率条件**：`h ≥ 1/(1-p_f)`（≈ `1+H·p`，需 H 倍余量）。余量不足时
  有效新帧率 `f_B = f0·(1-p_f)`。
- **内存开销**：上游须保留整帧以备重发：**`ΔM_B = F`**（双缓冲 `2F`）。
- **恢复时延**：`≈ T_f`（整帧）+ 往返。

### 策略 C — 整帧丢弃（不重采）
坏帧直接丢，等下一帧；这是“无恢复”基线。

- **内存开销**：**`ΔM_C = 0`**（不超过流水线已有的工作行缓存）。
- **有效帧率**：`f_C = f0·(1-p_f)`，坏帧**永久丢失**（运动卡顿/撕裂）。
- **恢复时延**：至多 `T_f`（等下一帧）。

## 3. 关键对比与量化边界

定义往返跨度比 **`ρ = D/H`**（往返窗口行数 / 整帧行数），它把三块差异统一起来：

| 指标 | A 行级 | B 整帧重传 | C 整帧丢弃 | A 相对优势 |
|---|---|---|---|---|
| 额外内存 | `D·L` | `F (=H·L)` | `0` | `ΔM_A/ΔM_B = ρ` → **省 `H/D` 倍** |
| 维持满帧率所需余量 | `1/(1-p)≈1+p` | `1/(1-p_f)≈1+Hp` | 不适用（丢帧） | 余量需求 **少 ≈H 倍** |
| 永久丢帧率 | 0（坏行可恢复） | 0 | `p_f≈Hp` | A/B 无永久损失，C 有 |
| 恢复时延 | `D·T_l` | `H·T_l` | `≤H·T_l` | **省 `H/D` 倍** |

**核心结论（可证伪命题，对应 §3）**：行级选择性重采集严格优于整帧策略，当且仅当

1. **往返窗口条件** `ρ = D/H ≪ 1`：往返跨越的行数远小于整帧行数；此时内存与恢复
   时延同时缩小为整帧重传的 `ρ` 倍。
2. **可承受重发条件** `p` 足够小，使 `h ≥ 1/(1-p)` 在链路余量内可满足；此时行级以
   `≈1+p` 的微小余量维持满帧率，而整帧重传需 `≈1+Hp`。

反向边界（行级**不**划算的区域）：当 `D → H`（往返时延接近整帧时间，例如上游无行缓存、
只能整帧搬运）或 `p` 大到 `Hp` 量级的重传开销已被余量吸收时，行级的内存与时延优势消失，
此时整帧策略实现更简单、等价更优。

**代表数值**（1920×1080 RAW10，`D=8`）：
- 内存：`ΔM_A = 8×2400 = 18.75 KiB` vs `ΔM_B = 2.47 MiB` → **约 135× 更省**。
- 恢复时延：`8·T_l ≈ 247 µs` vs `T_f ≈ 33.3 ms` → **约 135× 更短**。
- 满帧率余量：行级在 `p=10⁻³` 时只需 `h≈1.001`；整帧重传需 `h≈1+1080·10⁻³≈2.08`。

## 4. 与后续任务的接口

- **T2（行级闭环）**：camera model 的上游行缓存深度直接取 `D`；仿真需展示
  “出错→定位→请求→重采→恢复”，验证 `f_A=f0`、`ΔM_A=D·L`、恢复时延 `=D·T_l`。
- **T3（基线对比）**：脚本已实现 A/B/C 三条曲线，TB 对比实验复用同一组 `p(BER)`
  扫描点，把仿真测得的有效帧率/内存/时延叠加到本模型曲线上验证一致性。
- **T4（设计空间扫描）**：扫 `D`（往返窗口/上游缓存深度）与 `p`，定位
  `ρ = D/H` 的“划算/不划算”交叉边界。
- **T5（单向链路可行性条件）**：本模型条件 1、2 即为“单向链路本无重传、但补出行级
  恢复所需的前提”——上游缓存 `≥ D·L`、链路余量 `h ≥ 1/(1-p)`。

## 5. 假设与边界声明（答辩防守）

1. 模型针对**单向 CSI-2 RX**：标准链路无 ACK/NACK，重采集成立的前提是“上游具备可控
   缓存/可控图像源”，已在条件 1、2 显式列出，不声称标准 sensor 一定支持。
2. `p` 为 ECC/CRC 后的残余单行失效概率，假设行间错误独立；突发错误下 `p_f` 需按突发
   长度修正（留作 T4 敏感性分析）。
3. 重发自身也可能再错，已用几何分布 `1/(1-p)`、`1/(1-p_f)` 计入期望传输次数。
4. 内存项只计“为支持重发而额外保留”的缓存，不含流水线本就存在的工作缓存。
