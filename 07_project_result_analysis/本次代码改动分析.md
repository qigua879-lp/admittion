# 本次代码改动分析

## 1. 改动背景

这次更新主要是针对工程验证深度和 Vivado 时序结果做增强。原来工程已经能证明主链路功能闭环，但还有两个比较明显的短板：

1. `timing_cdc_v1` 中 Vivado 把 `clk_byte -> clk_sys` 的 CDC 路径当作普通同步路径分析，导致全局 routed WNS 为负。
2. AXI 写通路之前更多验证的是握手和 live pixel 输出，还缺少“写入后从存储读回再对比”的闭环证据。

## 2. RTL 改动分析

| 文件 | 改动内容 | 我理解的作用 |
| --- | --- | --- |
| `rtl/buffer/async_fifo.sv` | 给 Gray pointer 同步寄存器添加 `ASYNC_REG`、`SHREG_EXTRACT=NO` | 告诉 Vivado 这些寄存器是 CDC 同步器，不要随意优化成移位结构 |
| `rtl/top/mipi_csi2_capture_top.sv` | 给配置同步、lane error toggle、resync toggle 的跨域寄存器添加同类属性 | 让顶层 CDC 边界更明确，方便工具分析 |
| `rtl/axi/axi_write_null_slave.sv` | 增加内部 memory、byte 写入、WSTRB 支持、burst 地址递增、overflow response | 让 AXI sink 从“空接收器”变成可检查写入内容的仿真/FPGA 验证模型 |
| `fpga/xdc/top_constraints.xdc` | 新增 `set_clock_groups -asynchronous`，把 `clk_byte` 和 `clk_sys` 分为异步组 | 配合 FIFO 和同步器结构，避免把合法 CDC 路径误判成同步时序违例 |

这些改动没有改变外部顶层接口，也没有改变主要数据流协议。

## 3. 新增验证内容

| 新增项 | 文件 | 结果 |
| --- | --- | --- |
| AXI memory closure TB | `tb/tests/tb_fpga_wrapper_axi_mem_closure.sv` | 1/2/4 lane 均完成 RAW8 写入后 memory readback scoreboard |
| AXI memory lane regression 脚本 | `scripts/run_axi_mem_lane_regression.ps1` | 自动扫 `LANE_NUM=1/2/4` 并生成结果文档 |
| RAW8 soak metrics TB | `tb/tests/tb_fpga_wrapper_raw8_soak_metrics.sv` | 1/2/4 lane 均完成 32 帧、8 行/帧；lane2 另有 48 帧样例 |
| RAW8 soak/throughput 脚本 | `scripts/run_raw8_soak_throughput_sweep.ps1` | 自动采集吞吐、FIFO 峰值、stall 和 burst/beat 数 |

## 4. 新增结果摘要

### AXI memory closure

| lane | exp pixels | act pixels | aw bursts | w beats |
| ---: | ---: | ---: | ---: | ---: |
| 1 | 4 | 4 | 1 | 1 |
| 2 | 4 | 4 | 1 | 1 |
| 4 | 2 | 2 | 1 | 1 |

这说明当前 AXI 写通路不只是完成握手，而是能把像素写入内部 memory，并通过读回后 scoreboard 验证。

### RAW8 soak / throughput

| case | lane | frames | total pixels | pix/byte-clk x1000 | pix/axi-clk x1000 |
| --- | ---: | ---: | ---: | ---: | ---: |
| `lane1_soak_s0` | 1 | 32 | 1024 | 98 | 123 |
| `lane2_soak_s0` | 2 | 32 | 1024 | 106 | 133 |
| `lane4_soak_s0` | 4 | 32 | 1024 | 106 | 133 |
| `lane2_soak_long_s0` | 2 | 48 | 1536 | 106 | 133 |

这说明 lane1/lane4 的证据从最小 smoke 扩展到了多帧多行连续流。

### timing_cdc_v2

| 指标 | 数值 |

## 5. 仍需注意的问题

| 问题 | 说明 |
| --- | --- |
| 真实板级 signoff 仍未最终完成 | 实验版 `LOC/IOSTANDARD/IO delay` 已清掉 `NSTD-1/UCIO-1` 并生成 bitstream；真实上板还缺原理图 pinout、bank VCCO 和 timing closure |
| lane1/lane4 错误注入还不足 | 虽然有 soak 和 AXI readback，但 CRC/ECC/resync 压力测试仍主要在 lane2 |
| AXI memory closure 目前是单帧 | 读回闭环关注单帧写正确性，长时间压力由 soak/stress TB 补充 |
| 真实 DDR 未接入 | 当前 AXI sink 是验证模型，不等于真实 DDR 控制器 |

## 6. 更新后的交付口径

我建议现在对外汇报时使用下面这个口径：

```text
本工程已完成 MIPI CSI-2 数字图像采集前端的 RTL 主体设计、系统级 testbench、错误注入、AXI 写入读回验证和多 lane RAW8 soak 覆盖。Vivado timing_cdc_v2 结果显示，在当前占位时钟约束和 CDC 约束下 routed timing 已满足；board_io_v1 又证明实验版 LOC/IOSTANDARD/IO delay 下 bitstream DRC 已清零并可生成 bit 文件。但真实板级 pinout 签核、真实 DDR 集成和 ASIC signoff 仍属于后续工作。
```
