# 单向 MIPI CSI-2 链路上行级重采集的可行性条件（T5）

> 创新点第三根支柱（**条件**）。T1 给量化边界、T2 给机制、T3 给对比；本文回答最关键
> 也最有学术味的问题：**标准 CSI-2 高速数据链路是单向的、本无 ACK/NACK 重传，那么
> "行级选择性重采集"在什么前提条件下才成立？** 给出形式化的可行性条件与退化边界。
> 对应交接文档 §4 任务 T5、§3 命题中的"条件"一词。

---

## 1. 问题陈述与守红线

标准 MIPI CSI-2 的高速数据通道（D-PHY/C-PHY 上的 lane）是**单向**的：sensor→RX，
无反向 ACK/NACK，**因此不能声称"CSI-2 实现了自动重传"**。

本文的命题不是"给单向链路加重传"，而是：

> 在**保留高速数据链路单向性**的前提下，借助系统中**已存在的低速控制反向通道**
> （CCI / I²C，本就用于 sensor 配置）传回"重采请求 + 行定位"，并要求上游是
> **可控、可重发的图像源**，则可在满足下列条件时补出**行级**恢复能力。

即：链路仍单向，恢复能力来自"低速反向请求 + 可控源"的系统性补充，而非链路本身。

---

## 2. 系统模型与往返时延

一次行级重采集的时间链（坏行 L 发生在第 k 行）：

| 阶段 | 时延 | 说明 |
|---|---|---|
| 检出 | `τ_det` | line_end 后算完 CRC/ECC、`err_classifier` 锁存上下文、`retry_req` 拉高 |
| 反向请求 | `τ_bc` | 经 CCI/I²C 等低速通道把 (frame, line) 送达上游 |
| 源响应 | `τ_src` | 上游定位该行、发起重发 |
| 重发传输 | `τ_tx,line` | 重发一行经数据链路到达 RX 并写回 |

往返窗口 `τ_rt = τ_det + τ_bc + τ_src`（不含重发行本身的传输）。定义

> **D = ⌈τ_rt / T_l⌉** = 往返期间数据链路继续推进的"行数"。

D 是贯穿 T1/T3/T5 的核心量：T1 的内存/时延边界、T3 实测的 buffer_lines、本文的缓存
条件都以 D 表达。

---

## 3. 可行性的四个条件

### 条件 C1 — 上游行缓存深度
上游可控源必须**保留第 k 行直到重采请求到达**。在请求到达前，源已发出 D 行，故源端
行缓存深度需

> **`B_src ≥ D + 1`**，对应源端额外内存 `≈ (D+1)·L ≈ D·L`。

这正是 T1 模型里 A 策略的 `ΔM_A = D·L`，也与 T3 实测 `buffer_lines=D` 对应。

### 条件 C2 — 时间窗 / 活性（liveness）
请求必须在该行被源覆盖/丢弃前到达：

> **`τ_det + τ_bc ≤ B_src · T_l`**

即"检出 + 反向通道时延"必须落在源缓存能覆盖的时间窗内。这把 C1 的空间约束（缓存多大）
与反向通道速度（请求多快到）联系起来。

### 条件 C3 — 链路带宽余量
重发的行要占用数据链路时间，不能压垮标称帧率（沿用 T1）：

> **`h ≥ 1/(1-p)`**（≈ 1+p，余量极小即可）

其中 p 为单行残余失效概率，h 为链路带宽余量系数。注意整帧重传需 `h ≥ 1/(1-p_f) ≈ 1+Hp`，
即行级对余量的需求约小 H 倍——这是"为什么单向链路下行级才补得起、整帧补不起"的定量原因。

### 条件 C4 — 帧内行索引与幂等写回
RX 必须把**帧内（frame-relative）行索引**传给源并用于写回，确保：
(a) 源重发的是正确的行；(b) writer 把重发行**恰好一次**覆盖到该行的帧缓冲槽。
`recapture_writeback_ctrl` 的"按 retry_line_id 寻址、不推进计数器、不重复计数"提供了 (b)
的幂等性。**当前 RTL 的行号基转换仅首帧成立；多帧需真正的帧内行索引**（见 T2 遗留项），
这是 C4 在工程上尚未完全闭合的部分。

## 4. 可行性命题（形式化表述）

> **命题（行级重采集可行性）.** 设单向 CSI-2 数据链路外加一条低速反向通道，且上游为
> 可控可重发图像源。若 (C1) 源行缓存 `B_src ≥ D+1`、(C2) `τ_det+τ_bc ≤ B_src·T_l`、
> (C3) `h ≥ 1/(1-p)`、(C4) 存在帧内行索引且写回幂等，其中 `D = ⌈τ_rt/T_l⌉`，
> 则系统可恢复每个可丢弃的单行错误，**额外内存 ≈ D·L、恢复时延 = τ_rt = D·T_l，且与帧高 H 无关**。

### 退化边界（命题的反面，可证伪）
- **D → H**（源只能按帧粒度缓存/重发，如无行缓存的整帧 replay）：行级退化为**整帧重传**，
  内存/时延优势消失。
- **无反向通道 或 源不可重发**：C1/C2 不成立，只能**整帧丢弃（C 策略）**，无恢复。
- **p 大到 `h < 1/(1-p)`**：C3 不成立，重发压垮帧率，需先降分辨率/帧率或丢弃。

这三条退化边界正好对应 T3 的三种基线，使命题"在 ρ=D/H≪1 且条件满足时行级严格更优"
可被实验证伪/证实。

---

## 5. 哪些源满足条件（工程落地）

| 源类型 | C1/C2(可重发+缓存) | 适用性 |
|---|---|---|
| 带帧/行缓存且支持重读命令的 sensor | ✅ | 直接满足 |
| DRAM/存储回放的图像源（replay） | ✅ | 满足，D 取决于读回延迟 |
| FPGA/桥接前端（本工程的可控源模型） | ✅ | 满足，仿真已验证 |
| 无缓存的纯流式 raw sensor | ❌ | 不满足 C1，只能丢帧 |

**反向通道**用系统已有的 CCI/I²C 即可：以 400 kHz I²C 传几字节的 (frame,line)，`τ_bc`
约数十 µs；在 1080p30 下 `T_l ≈ 30.9 µs`，故 **D 量级为 1–2 行**（与 T3 仿真实测 D=1 同阶）。
即典型工作点天然落在 `D ≪ H`（H=1080）的"行级划算"区，可行性条件容易满足。

## 6. 与其余任务的接口

- **T1**：C1=ΔM_A、C3=链路余量条件，本文把它们从"模型假设"升级为"系统可行性条件"。
- **T3**：实测 `buffer_lines=D(A)/H(B)`、`recovered`(C 不满足 C1/C2) 即三条退化边界的实证。
- **T4**：扫 D 与 p，定位 C1–C3 同时成立的可行域边界（ρ=D/H 的"划算/不划算"交叉）。
- **多帧推广**：C4 的帧内行索引是把闭环从单帧推到多帧的关键，列为下一个 RTL 工作项。

## 7. 诚实边界

1. 本文是**可行性条件的形式化**，C1–C3 已与 T1/T3 定量对齐；C4 的多帧幂等在 RTL 尚未完全闭合。
2. `τ_bc/τ_src` 取决于具体反向通道与源实现，本文给出量级估计（I²C 数十 µs），精确值需按
   选定器件标定（留作 T4 敏感性分析的输入）。
3. 守红线：全程**不声称标准单向 CSI-2 链路具备重传**；恢复能力明确归因于"低速反向通道 +
   可控可重发源"这一系统性前提。
